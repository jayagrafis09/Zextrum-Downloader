# Setup Guide for Zextrum Downloader Backend\n\n## Prerequisites\n\nBefore running the backend, make sure you have the following installed:\n\n### 1. Node.js (v16 or higher)\n\n**macOS:**\n```bash\nbrew install node\n```\n\n**Ubuntu/Debian:**\n```bash\ncurl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -\nsudo apt-get install -y nodejs\n```\n\n**Windows:**\nDownload from https://nodejs.org/\n\n### 2. yt-dlp\n\nyt-dlp is a YouTube downloader that extracts video metadata and downloads content.\n\n**macOS:**\n```bash\nbrew install yt-dlp\n```\n\n**Ubuntu/Debian:**\n```bash\nsudo apt-get install yt-dlp\n```\n\n**Windows (via Chocolatey):**\n```bash\nchoco install yt-dlp\n```\n\n**Or via pip (any platform):**\n```bash\npip install yt-dlp\n```\n\n### 3. FFmpeg\n\nFFmpeg is required for audio extraction and format conversion.\n\n**macOS:**\n```bash\nbrew install ffmpeg\n```\n\n**Ubuntu/Debian:**\n```bash\nsudo apt-get install ffmpeg\n```\n\n**Windows (via Chocolatey):**\n```bash\nchoco install ffmpeg\n```\n\n**Or download from:** https://ffmpeg.org/download.html\n\n---\n\n## Installation Steps\n\n### 1. Clone the Repository\n\n```bash\ngit clone https://github.com/jayagrafis09/Zextrum-Downloader.git\ncd Zextrum-Downloader/backend\n```\n\n### 2. Install Dependencies\n\n```bash\nnpm install\n```\n\n### 3. Setup Environment Variables\n\n```bash\ncp .env.example .env\n```\n\nEdit `.env` and update paths if needed:\n\n```env\n# Find FFmpeg path\nwhich ffmpeg      # macOS/Linux\nwhere ffmpeg      # Windows\n\n# Update .env\nFFMPEG_PATH=/usr/local/bin/ffmpeg\nFFPROBE_PATH=/usr/local/bin/ffprobe\n```\n\n### 4. Verify Installation\n\n```bash\n# Check yt-dlp\nyt-dlp --version\n\n# Check FFmpeg\nffmpeg -version\nffprobe -version\n```\n\n---\n\n## Running the Backend\n\n### Development Mode\n\n```bash\nnpm run dev\n```\n\nOutput:\n```\n­¤ÜĆ Zextrum Downloader API running on port 5000\n­¤ōŹ Environment: development\n­¤īÉ CORS origin: http://localhost:3000\n­¤ōü Download path: ./downloads\n­¤Ä¼ FFmpeg configured at: /usr/local/bin/ffmpeg\n```\n\n### Production Mode\n\n```bash\nnpm run build\nnpm start\n```\n\n---\n\n## Testing the API\n\n### 1. Health Check\n\n```bash\ncurl http://localhost:5000/api/health\n```\n\nExpected response:\n```json\n{\n  \"status\": \"ok\",\n  \"message\": \"Zextrum Downloader API is running\",\n  \"timestamp\": \"2026-06-30T00:00:00.000Z\",\n  \"version\": \"1.0.0\",\n  \"ffmpeg\": \"Ō£ģ Available\"\n}\n```\n\n### 2. System Info\n\n```bash\ncurl http://localhost:5000/api/info/system\n```\n\n### 3. Get Video Info\n\n```bash\ncurl \"http://localhost:5000/api/info?url=https://www.youtube.com/watch?v=dQw4w9WgXcQ\"\n```\n\n### 4. Download Video\n\n```bash\ncurl -X POST http://localhost:5000/api/download/video \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"url\": \"https://www.youtube.com/watch?v=dQw4w9WgXcQ\",\n    \"quality\": \"720p\"\n  }'\n```\n\nSupported qualities: `highest`, `4k`, `1080p`, `720p`, `480p`, `360p`, `worst`\n\n### 5. Download Audio (MP3)\n\n```bash\ncurl -X POST http://localhost:5000/api/download/audio \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"url\": \"https://www.youtube.com/watch?v=dQw4w9WgXcQ\",\n    \"format\": \"mp3\"\n  }'\n```\n\nSupported formats: `mp3`, `m4a`, `wav`\n\n### 6. Download Subtitles\n\n```bash\ncurl -X POST http://localhost:5000/api/download/subtitles \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"url\": \"https://www.youtube.com/watch?v=dQw4w9WgXcQ\",\n    \"language\": \"en\"\n  }'\n```\n\n---\n\n## Troubleshooting\n\n### FFmpeg not found\n\n```bash\n# Find FFmpeg location\nwhich ffmpeg\n\n# Update .env\nFFMPEG_PATH=/path/to/ffmpeg\n```\n\n### yt-dlp errors\n\n```bash\n# Update yt-dlp\nyt-dlp -U\n\n# Test yt-dlp\nyt-dlp --dump-json \"https://www.youtube.com/watch?v=dQw4w9WgXcQ\"\n```\n\n### Port already in use\n\n```bash\n# Change port in .env\nPORT=5001\n```\n\n### Permission denied (downloads folder)\n\n```bash\n# Create downloads folder with correct permissions\nmkdir -p downloads\nchmod 755 downloads\n```\n\n---\n\n## API Documentation\n\n### Base URL\n```\nhttp://localhost:5000/api\n```\n\n### Endpoints\n\n| Method | Endpoint | Description |\n|--------|----------|-------------|\n| GET | `/health` | Check API status |\n| GET | `/info?url=<url>` | Get video information |\n| GET | `/info/system` | Check system dependencies |\n| POST | `/download/video` | Download video |\n| POST | `/download/audio` | Download audio |\n| POST | `/download/subtitles` | Download subtitles |\n\n---\n\n## Performance Tips\n\n1. **Close unnecessary apps** to free up system resources\n2. **Use lower quality** for faster downloads: `480p` or `360p`\n3. **Check bandwidth** before downloading large files\n4. **Monitor disk space** - downloaded files are stored in `./downloads`\n\n---\n\n## Next Steps\n\n- Setup frontend: `cd ../frontend && npm install && npm run dev`\n- Deploy backend to production\n- Configure database for persistent storage\n\n---\n\n**Need help?** Check the main [README.md](../README.md)\n
